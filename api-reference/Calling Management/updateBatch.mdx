---
title: "Update Batch"
description: "Modify batch metadata, scheduling windows, and retry behaviour."
openapi: "PUT /api/call/update-batch/{batchId}"
---

## Overview
Adjust batch settings when campaign requirements change—rename the batch, rotate calling numbers, tweak retry cadence, or configure CRM integrations. Updates take effect immediately for subsequent calls.

## Prerequisites
- Bearer token for the workspace admin.
- The `batchId` to modify.
- Desired scheduling and CRM configuration values.

## Request
<ParamField path="batchId" type="string" required>
Identifier of the batch you are updating.
</ParamField>
<ParamField header="Authorization" type="string" required>
Bearer token authorised to manage batches.
</ParamField>

<ParamField body="batchName" type="string">
New display name for the campaign.
</ParamField>
<ParamField body="callingNumbers" type="array[string]">
Updated list of phone IDs for outbound dialing.
</ParamField>
<ParamField body="retry" type="integer">
Maximum retry attempts per contact.
</ParamField>
<ParamField body="delay" type="integer">
Delay between retries, in minutes.
</ParamField>
<ParamField body="timezone" type="string">
Timezone used to evaluate calling windows (IANA identifier).
</ParamField>
<ParamField body="callingHour.start" type="string">
Start of the allowed calling window (HH:mm, 24-hour).
</ParamField>
<ParamField body="callingHour.end" type="string">
End of the allowed calling window (HH:mm, 24-hour).
</ParamField>
<ParamField body="allowedDays" type="array[string]">
Weekdays when calling is permitted.
</ParamField>
<ParamField body="CRMSchemaId" type="string">
CRM schema used to map incoming data.
</ParamField>
<ParamField body="IncomingDataCRMId" type="string">
CRM integration that supplies leads.
</ParamField>
<ParamField body="PushingDataCRMId" type="string">
CRM integration that receives call outcomes.
</ParamField>

<RequestExample>
```bash cURL
curl --location --request PUT '{{baseUrl}}/api/call/update-batch/6881d0057f3984aafeebeb78' \

  --data '{

    "type": "outbound",

      "64fd21b7e83f9d2c8a3e11b0",

    "callingHour": {

    "CRMSchemaId": "64fd21b7e83f9d2c8a3e11a9",
    "IncomingDataCRMId": "64fd21b7e83f9d2c8a3e11b0",
    "PushingDataCRMId": "64fd21b7e83f9d2c8a3e11b1"

```
</RequestExample>

## Response
<ResponseExample>
```json 200 OK
{
  "success": true,
  "batch": {
    "_id": "6881d0057f3984aafeebeb78",
    "batchName": "Customer Follow-up Batch",
    "timezone": "Asia/Kolkata",
    "retry": 3,
    "delay": 10
  }
}
```
</ResponseExample>

<ResponseField name="batch" type="object" required>
Contains the updated batch summary so you can confirm the applied settings.
</ResponseField>

## Error Handling
- **400 Bad Request** – Payload fails validation, e.g., invalid calling window or negative retry counts.
- **401 Unauthorized** – Bearer token missing or expired.
- **404 Not Found** – Batch not found or does not belong to the workspace.
- **500 Internal Server Error** – Update failed; retry or contact support.

<Tip>
Chain this endpoint with `GET /api/call/get-batch/{batchId}` to verify the configuration before resuming dialing.
</Tip>
