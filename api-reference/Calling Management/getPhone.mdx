---
title: "Get Phone Number"
description: "Retrieve the configuration for a provisioned outbound phone number."
openapi: "GET /api/call/get-phone/{phoneId}"
---

## Overview
Use this endpoint to inspect the telephony provider, credentials, and status tied to a specific outbound number. It is useful when auditing integrations or confirming which agent owns a number.

## Prerequisites
- Bearer token for a workspace admin.
- The `phoneId` returned when you created or imported the number.

## Request
<ParamField path="phoneId" type="string" required>
Identifier of the phone record you want to inspect.
</ParamField>

<ParamField header="Authorization" type="string" required>
Bearer token representing an approved administrator.
</ParamField>

<RequestExample>
```bash cURL
curl --location '{{baseUrl}}/api/call/get-phone/64e5f12345abcdef87654321' \
  --header 'Authorization: Bearer {{authToken}}'
```
</RequestExample>

## Response
<ResponseExample>
```json 200 OK
{
  "success": true,
  "phone": {
    "_id": "64e5f12345abcdef87654321",
    "userId": "64e5f12345abcdef12345678",
    "phoneNumber": "+15559876543",
    "integration": "plivo",
    "isActive": true,
    "createdAt": "2025-03-04T19:16:09.610Z"
  }
}
```
</ResponseExample>

<ResponseField name="integration" type="string">
Telephony provider connected to the number, for example `plivo`.
</ResponseField>
<ResponseField name="isActive" type="boolean">
Reflects whether the number is available for campaigns.
</ResponseField>

## Error Handling
- **404 Not Found** – `phoneId` is unknown or was deleted.
- **500 Internal Server Error** – Nexus could not fetch the phone record.

<Note>
Use this endpoint before updating or deleting to confirm you are operating on the correct phone number.
</Note>
