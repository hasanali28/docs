---
title: "Update Phone Number"
description: "Refresh the telephony provider credentials or number assigned to a workspace."
openapi: "PUT /api/call/update-phone/{phoneId}"
---

## Overview
Update phone records when you rotate provider credentials, migrate to a new telephony partner, or repurpose a number. The endpoint overwrites the stored credentials and keeps the number linked to existing batches.

## Prerequisites
- Bearer token for an admin user.
- The `phoneId` of the number you want to update.
- New provider details and credentials ready.

## Request
<ParamField path="phoneId" type="string" required>
Identifier of the phone configuration you want to update.
</ParamField>

<ParamField header="Authorization" type="string" required>
Bearer token representing the workspace admin.
</ParamField>

<ParamField body="phoneNumber" type="string" required>
Updated outbound number in E.164 format.
</ParamField>

<ParamField body="integration" type="string" required>
Provider slug such as `plivo` or `twilio`.
</ParamField>

<ParamField body="authToken" type="string" required>
Latest provider API token. Stored securely by Nexus.
</ParamField>

<ParamField body="accSId" type="string" required>
Provider account SID or equivalent identifier.
</ParamField>

<RequestExample>
```bash cURL
curl --location --request PUT '{{baseUrl}}/api/call/update-phone/64e5f12345abcdef87654321' \
  --header 'Authorization: Bearer {{authToken}}' \
  --header 'Content-Type: application/json' \
  --data '{
    "phoneNumber": "+15559876543",
    "integration": "plivo",
    "authToken": "updated_auth_token",
    "accSId": "updated_account_sid"
  }'
```
</RequestExample>

## Response
<ResponseExample>
```json 200 OK
{
  "success": true,
  "phone": {
    "_id": "64e5f12345abcdef87654321",
    "phoneNumber": "+15559876543",
    "integration": "plivo",
    "isActive": true
  }
}
```
</ResponseExample>

<ResponseField name="phone" type="object" required>
Updated phone record reflecting the new credentials.
</ResponseField>

## Error Handling
- **400 Bad Request** – Payload missing required fields or contains invalid data.
- **404 Not Found** – Phone record does not exist or is owned by another workspace.
- **500 Internal Server Error** – Update failed due to provider validation issues; retry with new credentials.

<Warning>
Updating provider credentials while a batch is active can interrupt live calls. Pause active campaigns before rotating tokens.
</Warning>
