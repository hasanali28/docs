---
title: "Get Agent"
description: "Fetch a specific agent and review its configuration."
openapi: "GET /api/agents/{id}"
---

## Overview
Use this endpoint to inspect the configuration of an individual agent, including conversation settings, workflow assignments, and status. It is typically used in admin dashboards when viewing agent details.

## Request
<ParamField path="id" type="string" required>
Agent identifier returned when the agent was created.
</ParamField>

<ParamField header="Authorization" type="string" required>
Bearer token with permission to manage agents.
</ParamField>

<RequestExample>
```bash cURL
curl --location '{{baseUrl}}/api/agents/684135442bc2d87ddfe0697f' \
  --header 'Authorization: Bearer {{authToken}}'
```
</RequestExample>

## Response
<ResponseExample>
```json 200 OK
{
  "message": "Agent retrieved successfully",
  "agent": {
    "_id": "684135442bc2d87ddfe0697f",
    "name": "Support Specialist",
    "status": "active",
    "workflowId": "68a45da7d4b3035aee9cb690",
    "conversationType": "multi_prompt"
  }
}
```
</ResponseExample>

<ResponseField name="agent" type="object" required>
Complete agent record including metadata, call settings, and workflow bindings.
</ResponseField>
<ResponseField name="agent.ambientNoise.enable" type="boolean">
Indicates whether background audio is injected during calls.
</ResponseField>

## Error Handling
- **404 Not Found** – No agent exists for the supplied `id`.
- **401 Unauthorized** – Bearer token missing or lacks the required scope.
- **500 Internal Server Error** – Unexpected failure retrieving the agent; retry or escalate.

<Tip>
Call `GET /api/agents/ambient-noise/options` when you need to display valid ambient-noise presets alongside this detail view.
</Tip>
